How it works

The model plans. Code renders. A human taps publish.

01

Generate

Images and clips via your own OpenAI or Gemini keys — no credit meters, no markup. A built-in mock provider keeps every workflow running before you add any keys.

02

Render

Declarative JSON plans compile to exact ffmpeg invocations: clips, trims, burned-in captions, music beds, brand kits. Same plan in, same pixels out.

03

Draft

Posts and threads become reviewable files, validated against each platform's real limits — grapheme counts, image sizes, media rules — before anything leaves your machine.

04

Publish

One confirm posts to Bluesky, Mastodon, or X — including reply-chained threads — and reports per-platform results honestly. Nothing ever auto-posts.

Deterministic by design

When AI output drifts, the fix is moving it into code.

// the agent writes this…
{
  "clips": [
    { "kind": "color", "color": "#101014", "durationSec": 2 },
    { "kind": "image", "asset": "ast_37d575", "durationSec": 3 }
  ],
  "overlays": [{ "text": "broll", "preset": "title" }],
  "captions": [
    { "text": "rendered by code, not vibes",
      "startSec": 0.4, "endSec": 2.0 }
  ],
  "aspect": "9:16"
}
// …broll compiles it to one exact ffmpeg command.

Safety model

Built so an agent can't embarrass you.

  • Draft-first, always. Nothing is published that isn't first a reviewable JSON file on disk.
  • Hard confirm gate. publish_post requires confirm: true — agents are instructed to get your explicit go-ahead.
  • No silent truncation. Constraint violations block publishing with instructions; your text is never auto-trimmed.
  • No double-posting. Retrying a partly-published draft refuses platforms that already succeeded.
  • Your keys stay yours. Read from env or ~/.broll/.env, sent only to their own vendor, never logged, never proxied.
