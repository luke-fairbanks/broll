what it does

fifteen tools. four verbs.

  • generate_image / _video

    your OPENAI_API_KEY or GEMINI_API_KEY, sent only to its own vendor. a keyless mock keeps every workflow runnable before you add keys.

  • render_video

    the agent writes a declarative json plan — clips, trims, burned-in captions, music bed, aspect — and broll compiles it to one exact ffmpeg invocation. same plan in, same pixels out.

  • render_carousel

    branded slide sets where layout, type scale, accent bar, page numbers are code. slide 40 matches slide 1, forever.

  • extract_frame

    pulls a png so the agent can look at its own render before showing you. agents that check their work post less garbage.

  • create_post_draft / _thread_draft

    posts become reviewable json files, validated against each platform's real rules — bluesky's 300 graphemes and 1MB blobs, x's urls-count-as-23 — before anything leaves the machine.

  • publish_post

    hard-requires confirm:true. publishes to a subset if you want, reports per-platform results, refuses to double-post on retry.

safety model

rules the code can't break.

  1. draft-first, always. nothing is published that isn't first a reviewable file on disk.
  2. hard confirm gate. publish_post requires confirm:true — agents are instructed to get your explicit go-ahead, every time.
  3. no silent truncation. constraint violations block publishing with instructions. your text is never auto-trimmed.
  4. no double-posting. retrying a partly-published draft refuses the platforms that already succeeded.
  5. your keys stay yours. read from env or ~/.broll/.env, never logged, never proxied, no markup.

these are numbered non-negotiables in the repo's CLAUDE.md — the file the agent itself is bound by when it works on this codebase.
